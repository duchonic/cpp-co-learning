# Class and Struct - `class` and `struct` are mostly the same thing - Classes have all the members private by default, in a `struct` they are public by default. - Rule of thumb? Prefer classes when you have functions inside of it.
# Defining a class in the file `rectangle.cpp` we have: - the definition of the `Rectangle` class. - the implementation of the `Rectangle` class. - the `main()` function which is the entry point for the program, creates a `rectangle` object, sets its width and height and reads its area.
~~~.cpp #include using namespace std; class Rectangle { public: Rectangle() {}; void setWidth(int w) {width = w;}; void setHeight(int h) {height = h;}; int getArea(); private: int width{0}; int height{0}; }; int Rectangle::getArea() { return height * width; } int main() { Rectangle rectangle{}; rectangle.setWidth(10); rectangle.setHeight(5); cout << "Area: " << rectangle.getArea() << endl; return 0; } ~~~
# "Running the class" in the file `rectangle.cpp` we have: - create a `build/` directory in `01-rectangle` - `cd build/` - `cmake ..` - `make` - `./rectangle`
